# MyPi

**MyPi 是一个电子画框管理系统，专注于水墨屏画面的自动化渲染与调度更新。** 它可以根据预设的时间规则，将天气、日历、待办事项等不同内容自动推送到屏幕上。

## 主要特性

1. **模板化扩展**
   支持通过后端代码开发接入各类「画面模板」（如每日天气、每日寄语等），并在 Web 控制台中以可视化的卡片形式提供管理。

2. **灵活的场景调度**
   可基于模板创建带有特定时间规则的「场景」。例如：设置场景 1 在每天早 7 点渲染“每日天气”上屏，场景 2 在工作日晚 8 点渲染“每日寄语”上屏。

3. **模板复用与多实例**
   同一个画面模板可被实例化为多个独立运行的场景。例如，基于“日历模板”可同时创建“工作日早 10 点更新”和“周一下午 1 点更新”两个互不干扰的场景配置。

4. **自动化任务编排**
   框架在后台统一接管所有场景的时间线。系统会自动处理并发冲突、合并排队，并在最合适的时机平滑、稳定地刷新屏幕，无需人工干预。

## 关键概念

为了帮助您快速理解本项目，我们对核心概念进行了统一：

| 中文 | 标识 | 说明 |
|------|------|------|
| **模板** | `Template` | 面向开发者和用户统一的概念，指代画面内容的定义与实现机制（如“每日天气”排版）。 |
| **场景** | `Scene` | 指代“模板”+“参数”+“调度时间”的组合实例。在 Web 控制台对应一张设定好刷新规则的卡片。 |
| **画框** | `Wall` | 内部抽象的核心域，代指最终的输出目标与整个系统（如 `WallState` 为画框状态）。 |
| **编排器** | `Orchestrator`| 核心系统组件，负责将多个场景按时间线进行排队和防冲突处理。 |
| **渲染管线** | `Pipeline` | 核心系统组件，负责取模板、出图、调校并推送到屏幕的流水线。 |

## 技术栈

- **后端**: Python 3.11+, Flask 3, Pydantic v2, APScheduler, Pillow
- **前端**: Vite, React, TypeScript
- **存储**: 无数据库依赖，基于本地 JSON/JSONL 文件持久化

## 仓库目录结构

```text
MyPi/
├── server/               # 后端服务 (Python 3.11+)
│   ├── api/              # RESTful API 路由层
│   ├── app/              # Flask 应用构建与入口
│   ├── display/          # 屏幕输出适配层
│   ├── domain/           # 核心业务模型与对齐逻辑
│   ├── orchestrator/     # 场景时间线编排器
│   ├── pipeline/         # 渲染与上屏管线
│   ├── renderers/        # 画面模板系统
│   │   └── templates/    # 各种预设或自定义模板实现
│   └── storage/          # 配置与状态的本地持久化
├── web/                  # 官方 Web 控制台 (React + Vite)
│   ├── src/components/   # UI 组件 (包含场景管理、画框预览等)
│   └── src/lib/          # 前端核心业务逻辑与工具
├── demo/                 # 画框演示与实验控制台
└── .cursor/              # 编辑器规则与 Agent 技能
```

## 快速开始

### 1. 准备后端环境

后端基于 Python 构建，建议使用 Python 3.11+。

```bash
cd server
python3 -m venv .venv
source .venv/bin/activate          # Windows PowerShell: .\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### 2. 准备前端环境

```bash
cd web
npm install
```

### 3. 本地启动服务

开发时需要开启**两个终端**分别运行前后端，Vite 会自动将 `/api` 请求代理到后端。

| 终端 | 目录 | 启动命令 | 访问地址 |
|------|------|------|------|
| **后端** | `server/` | `PYTHONPATH=. python _dev_serve.py` | 仅供 API 调用：`http://127.0.0.1:5050` |
| **前端** | `web/` | `npm run dev` | 控制台：**`http://localhost:5173/`** |

成功启动后，在浏览器打开 `http://localhost:5173/` 即可看到 Web 控制台。

> **提示**：如果想体验演示版界面，可以进入 `demo/` 目录同样执行 `npm install && npm run dev`。
